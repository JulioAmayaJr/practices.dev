Bases de Datos Relacionales
En una base de datos relacional, la información se organiza en tablas que están relacionadas entre sí. Estas relaciones se establecen mediante llaves primarias y llaves externas (o foráneas).
Tablas
  • Las tablas son la estructura básica en una base de datos relacional. Cada tabla tiene columnas que representan los atributos de los datos y filas que representan las entradas individuales de datos.
Llaves Primarias
  • Una llave primaria es un conjunto de columnas que identifica de forma única cada fila en una tabla. Se utiliza para garantizar la integridad de los datos y se define mediante la restricción PRIMARY KEY
Llaves Foráneas
  • Una llave foránea es una columna o conjunto de columnas en una tabla que hace referencia a la clave primaria de otra tabla. Se utiliza para establecer relaciones entre las tablas y se define mediante la restricción FOREIGN KEY
Relaciones
  • Las relaciones entre tablas se establecen mediante llaves primarias y foráneas. Por ejemplo, una tabla de "Pedidos" puede tener una llave foránea que hace referencia a la llave primaria de una tabla de "Clientes".

Ejercicio 1
En este ejercicio, vamos a crear una base de datos llamada "GestionTareas" que servirá como backend para una aplicación de gestión de tareas. Esta base de datos almacenará información sobre usuarios, proyectos y tareas, permitiendo a los usuarios organizar, asignar y monitorear el progreso de las tareas dentro de diferentes proyectos.
Pasos:
  • Crea una base de datos llamada GestionTareas.
  • Crea una tabla llamada Usuarios con las siguientes columnas (UsuarioID, Nombre, Email, Departamento)
  • Crea una tabla llamada Proyectos con las siguientes columnas (ProyectoID, Nombre, Descripcion, FechaInicio, FechaFin)
  • Crea una tabla llamada Tareas con las siguientes columnas (TareaID, ProyectoID, Descripcion, FechaCreacion, FechaLimite, UsuarioID, Estado)
Actividad
Se esta diseñando una base de datos para un hospital, el hospital tiene múltiples departamentos, cada uno con su propio personal médico y administrativo.Cada paciente tiene un historial médico que incluye información sobre diagnósticos, tratamientos y medicamentos suministrados. Los médicos pueden realizar múltiples consultas y procedimientos médicos para cada paciente.
A realizar:
  • Crear la base de datos.
  • Crear las tablas correspondiente.
  • Establecer las relaciones
  • Insertar al menos 2 datos por cada tabla
